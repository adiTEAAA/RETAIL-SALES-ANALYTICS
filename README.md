# Retail Sales Analytics Pipeline 🛒📊

An end-to-end data analytics pipeline that generates, cleans, analyzes, and prepares retail transaction data for visualization. This project demonstrates a complete data workflow integrating **Python**, **SQL**, and **Tableau**.

## 🚀 Overview

The goal of this project is to simulate a real-world data engineering and analytics environment. It takes raw, messy transaction data, cleans it, stores it in a relational database, performs complex SQL queries to extract business insights, and finally exports the structured data for interactive dashboarding in Tableau.

## 🛠️ Tech Stack

- **Python**: Pandas, Matplotlib, Seaborn (Data Generation, Cleaning, & Exploratory Data Analysis)
- **Database**: SQLite (Data Storage & Querying)
- **SQL**: Advanced querying for business intelligence
- **Visualization Tool**: Tableau (Dashboarding)

## 📁 Project Structure

| File / Folder | Description |
| :--- | :--- |
| `data_generation.py` | Generates a realistic synthetic dataset of retail transactions (`retail_transactions.csv`). |
| `data_cleaning_eda.py` | Cleans the raw data (handles duplicates, nulls) and performs Exploratory Data Analysis (EDA), saving plots to the `images/` directory. |
| `retail_sales.db` | A local SQLite database storing the cleaned transaction records. |
| `analysis_queries.sql` | Contains the core SQL queries used for business analytics. |
| `sql_analysis.py` | Executes the SQL queries against the database and prints the results. |
| `export_to_tableau.py` | Aggregates and exports specific datasets into CSV format, optimized for loading into Tableau. |
| `images/` | Directory containing EDA visualizations generated by Python. |

## 📊 Key Analytics & Insights

The pipeline executes SQL queries and Python visualizations to uncover the following insights:
1. **Monthly Revenue Trends**: Tracking overall business growth over time.
2. **Top-Performing Products & Categories**: Identifying which items drive the most revenue.
3. **Customer Segment Performance**: Analyzing purchasing behaviors across different customer demographics (e.g., Premium vs. Standard).
4. **Payment Method Popularity**: Understanding how customers prefer to transact.
